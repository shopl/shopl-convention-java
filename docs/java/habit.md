# 팀을 위한 개발 습관
본 문서는 필수적이지는 않지만 함께 개발을 해나가는 데 있어 서로에게 도움이 될 수 있는 추천사항들을 정리했습니다.

## TODO 주석 활용
구체적인 비지니스 로직을 작성하기 전 TODO 주석을 이용하여 각 개발의 개요를 구성하는 것을 추천한다.
TODO 주석의 사용할 때는 다음과 같이 작업자를 명시한다.

```java 
// TODO @harry 진행예정 업무
```
